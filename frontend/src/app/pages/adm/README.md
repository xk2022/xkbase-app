你這段 Sidebar 的「ADM 系統設定」其實是在做一個 **平台共用的後台治理（Governance）模組**：讓系統維運/管理者不需要改程式，就能調整「資料字典、參數、通知模板、審計追蹤」這類橫切能力。

下面我用「這模組給誰用 / 要放哪些功能 / 能做到什麼」來拆解，順便把你現在 v1~v3 的規劃合理化。

---

## 這個 ADM 模組主要是給誰用？

通常分三類角色（越往上權限越大）：

1. **系統管理員 / 平台維運（IT / DevOps / SRE）**

* 需要能快速調參數、開關功能、查日誌、定位問題
* 不想每次都找工程師改設定、重上版

2. **業務/營運管理者（Operations / Back-office）**

* 需要管理「字典值」（例如訂單狀態、車種、櫃型、費用項目…）
* 需要可控的通知模板（簡訊/Email/站內通知）讓對外訊息一致

3. **稽核 / 主管（Audit / Compliance / 管理層）**

* 需要「誰在什麼時候改了什麼」的可追溯性
* 需要關鍵操作的追查與報表（這就是 audit-log）

> 你把它放在「ADM 系統設定」是對的：這些都不是業務流程功能，而是平台管理/治理能力。

## 你現在 Sidebar 放的功能，定位是什麼？

### v1：Dictionary（字典檔）

**定位：** 平台共用的「下拉選單 / 枚舉值 / 分類碼表」管理
**能做到：**

* 新增/停用/排序字典項
* 多語系 label（可選）
* 維持前端下拉選單與後端碼表一致
  **典型使用者：** 營運、系統管理員
  **典型例子：**
* 訂單類型、訂單狀態、車種、櫃型、港口代碼、費用項目、客戶類型…

> 這是最值得先做的 v1：因為很多系統的下拉選單一開始都靠 hardcode，後面會痛。

---

### v2：Param（系統參數）

**定位：** 「可設定的系統行為」與「環境/租戶級設定」
**能做到：**

* Feature Toggle（例如：是否開放某流程、是否強制某欄位必填）
* 門檻值（超時分鐘數、提醒天數、預設分頁 size）
* 第三方整合設定（但敏感資訊要拆分成密鑰管理，不要直接明文）
  **典型使用者：** IT / 維運 / 產品管理
  **典型例子：**
* `ORDER_AUTO_ASSIGN=true/false`
* `DEFAULT_PAGE_SIZE=20`
* `VEHICLE_MAINTENANCE_REMIND_DAYS=30`

> v2 的價值是「不用上版也能調整策略」，但要做權限管控 + 變更紀錄。

---

### v3：Notification Template（通知模板）

**定位：** 對外/對內通知的內容「可控」且「可版本化」
**能做到：**

* 模板維護（Email/簡訊/站內通知）
* 變數插值（例如 `{orderNo}`, `{customerName}`）
* 啟用/停用、版本管理、預覽、測試發送（可選）
  **典型使用者：** 營運 / 客服主管 / IT
  **典型例子：**
* 訂單建立通知、指派司機通知、到港提醒、帳款提醒…

> v3 很常被業務需求推著走，做得好會大幅減少工程師「改字」的工單。

---

### v3：Audit Log（操作日誌）

**定位：** 「追責、稽核、除錯」的基礎建設
**能做到：**

* 查詢誰在什麼時間對什麼資源做了什麼操作（CRUD/登入/權限變更）
* 支援篩選（人、功能、資源、時間、結果 success/fail）
* 重要操作的不可否認性（至少能追溯）
  **典型使用者：** 稽核、主管、IT
  **典型例子：**
* 誰把某系統停用？誰把某角色權限加大？誰刪了一筆資料？

> 只要有 UPMS，Audit Log 幾乎一定會被需要，尤其你們有「權限管理」時。

---

## ADM 模組應該「左哪些功能」才合理？

你目前的四個方向很正確。我會建議 Sidebar 最終長這樣的資訊架構（按價值與使用頻率排序）：

* **字典檔 Dictionary（v1）** ✅ 必做
* **系統參數 Param（v2）** ✅ 很推薦
* **通知模板 Notification Template（v3）** ✅ 視你們是否有通知需求
* **操作日誌 Audit Log（v3）** ✅ 強烈建議

如果還要擴充（v4+）常見的會是：

* **排程/任務管理**（重送通知、補跑對帳、批次匯入）
* **檔案管理**（上傳的附件、匯入檔紀錄）
* **黑名單/白名單**（IP/帳號/客戶）

---

## 一句話總結：ADM 能做到什麼事？

> ADM 是「平台維運與營運自助化」：把原本需要工程師改程式/改設定的事情，變成後台可控、可審計、可回溯的管理能力。
